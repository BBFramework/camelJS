Camel.GetHttpRequest

Introduction

Camel.GetHttpRequest contains some functions about loading via http request methods.

Members

_suc

Hold the success callback.

How to use

  • var request = new Camel.GetHttpRequest();
  • request.addEventListener( 'ready', function()
  • {
  •     alert( 'OK' );
  • }, false);
  • console.log( request._suc );

_err

Hold the error callback.

How to use

  • var request = new Camel.GetHttpRequest();
  • request.addEventListener( 'error', function()
  • {
  •     alert( 'ERR' );
  • }, false);
  • console.log( request._err );

_xhr

Hold the XMLHttpRequest instance object.

How to use

  • var request = new Camel.GetHttpRequest();
  • request.Url( 'http://example.com/asset.json' );
  • console.log( request._xhr );

_m

Hold the request type. Default, it is given 'GET'.

How to use

  • var request = new Camel.GetHttpRequest();
  • console.log( request._m );

The return

Methods

addEventListener( e, callback, t )

Adds a event callback. It have two event types includes 'ready' and 'error'.

'error' type fires when the request meet an error.

'ready' type fires when the request is successful.

The param

  • e:string the string presents the event type.
  • callback:function the function has executed when the event fires.
  • t:boolean the trace flag.

How to use

  • var request = new Camel.GetHttpRequest();
  • request.addEventListener( 'ready', function()
  • {
  •     alert( 'OK' );
  • }, false);

The return

The void return.
